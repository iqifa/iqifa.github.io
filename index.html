杂项

部分Unity编辑器的知识，现在还不知道放在那里

    [SerializeFiled]

即Unity中的序列化的一种特性，Unity默认对public变量序列化使其能够在Inspector界面显示以及修改

对于private变量，想要在其他类中通过代码修改或者是在检视界面修改对应值，可以使用
[SerializeFiled]
    private string value;
    

而想要可以通过代码赋值，而又不想显示在编辑器中则使用
[HideInspector][SerializeFiled]
    private string vaule;


[HideInspector]
将变量从检视界面隐藏，也可用于public类型的变量

[DidReloadScripts]
在Unity重新加载脚本时执行

[Assembly]
C#程序集

c#中反射机制即使用此种实现，程序集中包含了类的所在位置，如此可以使得在运行时获得类的信息。也可以通过此种方法生成类的对象即反射。

Unity编辑器扩展

编辑器菜单

判断菜单项是否可用函数。

编辑器创建窗口(ScriptableWizard)

此窗口通过继承 ScriptableWizard 类进行，其中有函数

编辑器窗口(Window)

此窗口通过继承 EditorWindow 进行，与ScriptableWizard 不同的是，此窗口可以通过调用

OnGUI()

通过编写代码实现窗口布局，也通过

Window window = EditorWindow.GetWindow<Window>();
window.Show();

来显示窗口。

操作撤回

Unity的操作中会有部分修改需要撤回，比如删除某个东西或者移动某个东西，会使用到代码进行操作比如删除某个物体

而这种操作并不能撤回，对于编辑器而言是不合适的，所以会使用到

Undo

Undo.DestroyObjectImmediate(object);

这样就会保存执行的操作

数据保存

在编辑器扩展的窗口中可能需要保存上次修改时输入的数据，如使用ScriptableWizard修改数据，此处会使用到

EditorPrefs

除此之外，Json也是一种保存方法

编辑器中物体选择

Selection
编辑器扩展中，有时需要对选中的物体进行修改

GameObject[] gameObjects = Selection.gameObjects;
